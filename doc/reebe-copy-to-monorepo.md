# Moving Reebe into a pnpm + Turborepo Monorepo

Here's exactly what to do.

---

## Step 1 — Copy Reebe into `apps/`

```bash
cp -r /path/to/reebe apps/reebe
```

The pnpm workspace already globs `apps/*`, so no changes to `pnpm-workspace.yaml`.

---

## Step 2 — Add `apps/reebe/package.json`

Turborepo needs a `package.json` to treat it as a workspace member:

```json
{
  "name": "@bpmnkit/reebe",
  "version": "0.1.0",
  "private": true,
  "scripts": {
    "build": "cargo build --release -p reebe-server",
    "build:wasm": "wasm-pack build crates/reebe-wasm --target web --scope bpmnkit --out-dir ../reebe-wasm",
    "check": "cargo check --workspace",
    "test": "cargo test --workspace"
  }
}
```

The `--scope bpmnkit` flag makes wasm-pack emit `"name": "@bpmnkit/reebe-wasm"` in the generated `package.json`.

---

## Step 3 — Create the `apps/reebe-wasm/` stub

pnpm needs this directory to exist with a `package.json` before the first build, otherwise workspace resolution fails on fresh clone.

**`apps/reebe-wasm/package.json`** (commit this):
```json
{
  "name": "@bpmnkit/reebe-wasm",
  "version": "0.1.0",
  "description": "Reebe BPMN engine compiled to WebAssembly",
  "files": ["reebe_wasm_bg.wasm", "reebe_wasm.js", "reebe_wasm.d.ts"],
  "module": "reebe_wasm.js",
  "types": "reebe_wasm.d.ts",
  "sideEffects": false
}
```

wasm-pack will overwrite this with its own generated version on every build, but the fields above match what it produces so there's no conflict.

**`apps/reebe-wasm/.gitignore`** (don't commit the binary):
```
*.wasm
reebe_wasm.js
reebe_wasm_bg.js
reebe_wasm.d.ts
reebe_wasm_bg.wasm.d.ts
snippets/
```

---

## Step 4 — Update the justfile

In `apps/reebe/justfile`, update `build-wasm` to point to the monorepo-relative output:

```just
# Build the WASM playground package
build-wasm:
    wasm-pack build crates/reebe-wasm --target web --scope bpmnkit --out-dir ../reebe-wasm

# Check the WASM crate for compile errors (no wasm-pack needed)
check-wasm:
    cargo check -p reebe-wasm
```

---

## Step 5 — Add tasks to `turbo.json`

```json
{
  "tasks": {
    "build:wasm": {
      "dependsOn": [],
      "inputs": [
        "crates/**/*.rs",
        "crates/**/Cargo.toml",
        "Cargo.toml",
        "Cargo.lock"
      ],
      "outputs": ["../reebe-wasm/reebe_wasm*", "../reebe-wasm/snippets/**"]
    },
    "build": {
      "dependsOn": ["^build", "^build:wasm"]
    }
  }
}
```

This tells turborepo: any package whose `build` depends on `@bpmnkit/reebe-wasm` will wait for `build:wasm` to finish first, and the output is cached by the Rust source files as inputs.

> If you're on turbo v1, replace `"tasks"` with `"pipeline"`.

---

## Step 6 — Depend on it from other packages

In any `apps/*/package.json` or `packages/*/package.json`:

```json
{
  "dependencies": {
    "@bpmnkit/reebe-wasm": "workspace:*"
  }
}
```

Then in code:

```js
import init, { WasmEngine } from "@bpmnkit/reebe-wasm";

await init();
const engine = new WasmEngine();
```

---

## Step 7 — Root `.gitignore`

Add to the monorepo root `.gitignore`:

```
# Reebe WASM build artifacts (regenerated by wasm-pack)
apps/reebe-wasm/*.wasm
apps/reebe-wasm/*.js
apps/reebe-wasm/*.d.ts
apps/reebe-wasm/snippets/
# Rust build output
apps/reebe/target/
```

---

## First build after moving

```bash
pnpm install          # picks up the new workspace members
cd apps/reebe
wasm-pack build crates/reebe-wasm --target web --scope bpmnkit --out-dir ../reebe-wasm
cd ../..
pnpm build            # turborepo handles the rest
```

Or via turbo directly:

```bash
turbo run build:wasm --filter=@bpmnkit/reebe
```
